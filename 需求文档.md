# 「別急，先查證」反詐騙互動闖關遊戲 — 需求文檔

## 一、項目概述

一款面向海外留學生的反詐騙互動闖關遊戲，通過沉浸式劇情模擬真實詐騙場景，幫助玩家識別各類騙局套路。玩家選擇身份和地區後，經歷 6 個關卡的連環劇情，根據選擇累計數值，最終觸發不同結局。

---

## 二、核心玩法

### 2.1 遊戲流程
```
開始頁 → 選擇身份 → 選擇地區 → 開場介紹 → 6關闖關 → 結局 → 回顧頁
```

### 2.2 身份系統
- **中國大陸學生**：微信/支付寶/快遞/電商日常使用，面對語言壓力和身份焦慮
- **港澳台學生**：熟悉本地銀行、LINE 和學校電郵

### 2.3 地區地圖（5個）
| 地區 | 騙局主題 | 標籤 |
|------|---------|------|
| 中國大陸 | 快遞釣魚 + 假客服二次施壓 | 基礎關卡 |
| 港澳 | 假銀行短訊 + 釣魚網站 | 金融場景 |
| 台灣 | ATM 解除分期詐騙 | 高壓場景 |
| 歐美 | 假移民局電話詐騙 | 語言挑戰 |
| 東南亞 | 酒店付款連環釣魚 | 異地場景 |

### 2.4 關卡結構
- 每局 **6 關**（stage dots 顯示 6 個進度點）
- 每個地區 **1 種騙局劇情**（scenario），包含多個分支場景
- 場景類型：訊息通知、聊天、來電、電郵、網頁、核實結果、通關

### 2.5 敘事弧（6關對應）
```
鋪墊 → 試探/選擇分支 → 騙子加壓 → 查證/轉折 → 關鍵選擇 → 結局
```
- 選項不二元化（good/bad/mid 三種反饋）
- 騙子有自適應話術（對玩家的質疑有準備好的藉口）
- 正確選擇不立刻結束遊戲，繼續推進劇情
- 域名識別作為核心反釣魚機制

## 三、數值系統

### 3.1 基礎屬性
| 屬性 | 初始值 | 說明 |
|------|-------|------|
| 金錢 | 100 | 受騙會減少 |
| 警覺 | 50 | 識破能力 |
| 信任 | 50 | 對他人話語的信任度 |
| 壓力 | 20 | 越高越容易做錯決定 |
| 資訊 | 30 | 掌握的核實信息 |
| 本地熟悉度 | 50 | 對當地環境的了解 |
| 語言壓力 | 20 | 非母語環境壓力 |
| 身份焦慮 | 20 | 身份不安全感 |
| 社會壓力 | 20 | 社交壓力 |
| 風險分 | 0 | 越高越危險 |

### 3.2 身份/地區修正
- 不同身份 + 地區組合有初始數值加成/減成
- 在開場頁以 chips 形式展示

### 3.3 結局判定（8種）
| 結局 ID | 觸發條件 | 說明 |
|---------|---------|------|
| ending_deep_scammed | money ≤ 10 && riskScore ≥ 70 | 被騙慘了 |
| ending_high_risk_trap | riskScore ≥ 80 && information ≤ 30 | 高風險中招 |
| ending_anti_scam_expert | information ≥ 75 && alertness ≥ 70 && money ≥ 70 | 反詐達人 |
| ending_calm_verifier | information ≥ 65 && riskScore ≤ 35 | 沉住氣了 |
| ending_over_suspicious | trust ≤ 15 && information ≤ 40 | 太多疑 |
| ending_data_leak | riskScore ≥ 45 && money ≥ 70 | 資料洩露 |
| ending_small_loss | money < 70 && money > 10 | 小損失 |
| ending_safe_but_lucky | true（默認） | 運氣好 |

## 五、特殊機制

### 5.1 倒計時系統
- 僅在騙子來電等高壓場景觸發（speaker === 'scammer' 或 type === 'phone_call' && pressure === true）
- 10 秒限時選擇
- 時間到自動選擇風險最高的選項
- 最後 3 秒紅色警告動效

### 5.2 場景回退保護（localStorage）
- 每次選擇後自動保存完整遊戲狀態
- 刷新頁面後顯示「繼續上次進度」按鈕
- 遊戲結束後清除存檔

### 5.3 結局收集系統（localStorage）
- 跨遊戲會話持久保存已解鎖結局
- 結局頁顯示收集進度條 + 網格
- 起始頁可查看所有已解鎖/未解鎖結局

### 5.4 分數評級
| 分數 | 評級 | 圖標 |
|------|------|------|
| ≥ 200 | S | 🏆 |
| ≥ 150 | A | 🥇 |
| ≥ 100 | B | 🥈 |
| ≥ 60 | C | 🥉 |
| < 60 | D | 💀 |

## 六、UI/UX 要求

### 6.1 視覺風格
- 深色主題（#0D0D18 背景）
- 毛玻璃效果（backdrop-filter: blur）
- 霓虹光暈（紫色/粉色/綠色漸變）
- 微動畫（浮動、脈衝、漸變位移）

### 6.2 頁面結構（7個畫面）
1. **開始頁**：Logo + 標題 + 開始按鈕 + 繼續進度按鈕 + 結局收集入口
2. **身份選擇頁**：2 張身份卡片
3. **地區選擇頁**：5 張地區卡片
4. **開場頁**：身份/地區徽章 + 劇情介紹 + 屬性修正預覽 + 關卡預覽
5. **遊戲主頁**：側欄（狀態數值 + XP + 等級 + 進度點）+ 主區（場景卡片 + 選項）
6. **結局頁**：結局圖標/標題/描述 + 分數評級 + 數值 chips + 成就 + 反詐知識 + 結局收集進度
7. **回顧頁**：本局資料 + 詐騙紅旗 + 選擇分析 + 正確查證方式 + 最終數值

### 6.3 響應式
- 桌面：左右佈局（側欄 + 主區）
- 移動端（≤768px）：上下佈局，側欄折疊到頂部

---

## 七、技術棧

- 純前端：HTML + CSS + JavaScript（無框架）
- 數據存儲：localStorage（遊戲進度 + 結局收集）
- 無後端依賴，無需登錄

## 九、已知待完善項

1. **場景 ID 交叉引用校驗**：啟動時檢查所有 nextSceneId 是否有對應場景定義
2. **最大場景數保護**：超過 20 個場景強制觸發結局，防止死循環
3. **倒計時細化**：部分對話場景可加 pressure: false 跳過倒計時
4. **數值平衡**：8 個場景下 riskScore 累加速度需調整
5. **成就系統補充**：speedrun 改為「前 3 關全部正確」；增加「完整通關」成就
6. **身份/地區組合區分度**：不同身份遇到不同初始場景或對話差異
7. **音效與動效**：倒計時心跳音效、移動端震動反饋
